# RainDrop 2.0 UI/UX 개편 프롬프트

## 대주제

> **"채움으로 삶의 밀도를 기록한다"**

RainDrop은 macOS 집중 타이머 앱이다. 빗물이 양동이에 한 방울씩 차오르듯, 사용자의 집중 시간이 쌓여 하루의 밀도를 만든다. 2.0에서는 이 "채움"의 철학을 앱 전체 경험에 관통시킨다.

---

## 1. 현재 상태 (AS-IS)

### 앱 기본 정보
- **플랫폼**: macOS (SwiftUI + SPM, Xcode 프로젝트 아님)
- **아키텍처**: MVVM + DI Container (`AppContainer`), `@MainActor` 기반
- **창 크기**: 1040×700 고정
- **테마**: 다크/라이트 모드 지원 (`AppColors` 토큰 시스템)
- **현재 버전**: v1.4.0

### 현재 화면 구성

#### 메인 — TimerScreen (1040×700)
- **좌측 패널 (440px)**: 세션 목표, 타이머(58pt 모노스페이스), 일일 누적 프로그레스바, 시작/일시정지/중지 버튼, 버전 표시
- **우측 패널**: 동기부여 메시지(8초 순환), 비 파티클(Canvas, 40개), 구름 애니메이션, 양동이+물 차오르는 애니메이션, 스티커 꾸미기, 세션 완료 배너
- **헤더**: "RainDrop" 타이틀 + 잔액(🪣 코인) + 네비게이션 버튼들

#### 서브 화면들 (모두 sheet 모달)
| 화면 | 크기 | 역할 |
|------|------|------|
| HistoryScreen | 620×580 | 월별 히트맵 캘린더 + 세션 리스트 (segmented picker) |
| SettingsScreen | 420×320 | 목표시간, 집중확인, 양동이 스킨(6종), 패치노트 |
| ShopScreen | 500×480 | 이모지 스티커 구매 (4열 그리드) |

| SocialScreen | 500×600 | 로그인, 친구, 실시간 집중 상태, 랭킹 |

| PatchNotesView | 480×500 | 버전 히스토리 |

#### 현재 비주얼 요소
- **양동이 스킨 6종**: Wood → Dented Iron → Platinum → Gold → Diamond → Rainbow (누적 양동이 수로 언락)
- **비 파티클**: 40개, 캡슐 형태, 속도/크기/투명도 랜덤
- **구름**: 8초 수평 드리프트, 0.25~0.40 투명도
- **물 웨이브**: 2.5초 주기, 2레이어 사인파
- **동기부여 메시지**: 8초마다 전환
- **스티커**: 이모지 기반, 드래그 배치

#### 현재 게임화 시스템
- 세션 완료 → 양동이 코인 획득
- 코인으로 스티커 구매
- 양동이 누적 수로 스킨 언락
- 무한 모드: 양동이 순환, 무제한 타이머

#### 현재 네비게이션 플로우
```
앱 시작 → [온보딩 여부] → TimerScreen (허브)
  ├─ 히스토리 버튼 → HistoryScreen
  ├─ 설정 버튼 → SettingsScreen
  ├─ 상점 버튼 → ShopScreen
  ├─ 백색소음 버튼 → WhiteNoiseScreen
  ├─ 소셜 버튼 → SocialScreen
  └─ 메뉴바 → 타이머 상태/제어
```

## 2. 디자인 철학 (WHY)

### 핵심 메타포: "채움"
빗물 한 방울은 미미하지만, 양동이를 채우면 하루의 무게가 된다. RainDrop 2.0은 이 과정을 **보고, 느끼고, 되돌아보는** 경험으로 만든다.

### 감성 키워드
- **고요한 몰입** — 화려하지 않지만 계속 보고 싶은 화면
- **축적의 실감** — 오늘 얼마나 채웠는지 직관적으로 느낌
- **밀도의 시각화** — 숫자가 아닌, 물의 높이·색의 깊이·날의 무게로 표현
- **자연스러운 리듬** — 비 → 채움 → 비움 → 다시 비. 하루의 사이클

### 디자인 원칙
1. **물이 언어다** — 진행률은 %가 아니라 수위로, 달성은 숫자가 아니라 넘침으로
2. **밀도가 보인다** — 캘린더는 색의 농도로, 주간은 수위의 높이로, 연간은 파도의 크기로
3. **고요하되 살아있다** — 미세한 움직임(잔물결, 빗소리, 수면 반사)이 화면을 숨 쉬게 한다
4. **보상은 자연에서** — 코인이 아니라 날씨가 변하고, 양동이가 아니라 풍경이 진화한다

## 3. 개편 요구사항 (WHAT)

### 3-A. 메인 타이머 화면 개편

#### 레이아웃 재구성
- 현재: 좌(정보+제어) / 우(비주얼) 2분할
- **개편 방향**: "채움"이 화면의 중심이 되도록 재구성. 양동이와 물이 주인공이고, 타이머/제어는 이를 보조하는 구조로.

#### 비주얼 강화
- **물의 질감**: 현재 단순 사인파 → 더 자연스러운 물 표현 (잔물결, 수면 반사, 물방울 튀김)
- **비의 분위기**: 40개 파티클 → 씬 전체를 감싸는 빗줄기. 세션 진행도에 따라 비의 강도 변화
- **배경 깊이감**: 단색 그라데이션 → 시간/진행도에 따라 변하는 하늘(새벽→낮→저녁 또는 맑음→흐림→비)
- **넘침의 순간**: 양동이가 가득 찰 때의 연출을 특별하게. 물이 넘치고, 빗줄기가 잦아들고, 하늘이 개는 등

#### 타이머 표시
- 거대한 숫자 대신, 물의 수위 자체가 시간을 말하게 하되 정확한 시간은 작게 보조
- 세션 완료까지 남은 시간의 감각을 "양동이 상단까지 남은 공간"으로 직관화

#### 제어 버튼
- 시작/일시정지/중지의 3버튼 구조 유지하되, 화면 하단에 미니멀하게
- 타이머 진행 중에는 제어 UI를 최소화하여 몰입 극대화

### 3-B. 기록/회고 화면 개편 (HistoryScreen)

#### "밀도 캘린더"
- 현재: 5단계 색상 히트맵
- **개편 방향**: 각 날짜 셀이 작은 양동이(또는 물방울)이고, 그 안의 수위로 밀도를 표현
- 주간 단위로 보면 7개의 양동이가 나란히, 월간 단위로 보면 한 달의 밀도 흐름이 한눈에

#### 회고 경험
- 날짜 클릭 시 "그날의 양동이"를 크게 보여주며, 세션 타임라인을 시각적으로
- 주간/월간 요약: "이번 주 양동이 N개를 채웠어요" → 물이 차오르는 애니메이션으로 회고

### 3-C. 보상/성장 시스템 개편

#### 현재 문제
- 코인 → 이모지 스티커 구매는 "채움"과 연결 약함
- 양동이 스킨 6종은 단계감은 있지만 내러티브 부족

#### 개편 방향
- **환경 진화**: 집중 시간이 쌓이면 양동이 주변 환경이 변화 (풀 → 꽃 → 나무 → 숲 → 호수 등)
- **날씨 시스템**: 연속 집중일수에 따라 날씨 변화 (흐림 → 보슬비 → 소나기 → 무지개)
- **물의 색 변화**: 스킨 교체가 아니라, 집중 패턴에 따라 물의 색이 자연스럽게 진화
- 기존 스티커 시스템 유지하되, "채움"과 연결된 보상 추가

### 3-D. 백색소음 통합

- 현재: 별도 모달로 분리
- **개편 방향**: 메인 화면에 자연스럽게 녹이기. 비가 오면 소리도 함께. 토글은 유지하되 별도 화면 불필요

### 3-E. 온보딩 개편

- 현재: 4페이지 캐러셀 (정보 전달 위주)
- **개편 방향**: 첫 빗방울이 떨어지고, 양동이에 물이 차고, "이게 당신의 하루입니다"로 끝나는 체험형 온보딩. 설명이 아니라 경험.

### 3-F. 설정 화면

- 현재 Form 기반 유지 가능
- 스킨 선택 UI를 "환경 선택"으로 확장할 수 있도록 구조 고려

## 4. 기술 제약사항

- **macOS 전용** (SwiftUI, SPM 기반, Xcode 프로젝트 아님)
- **창 크기**: 고정 또는 최소 크기 지정 (현재 1040×700)
- **Canvas 기반 파티클**: 성능상 Canvas API 유지 권장
- **WKWebView**: 백색소음은 rainymood.com WebView 방식 유지 (라이프사이클 관리 완료)
- **Firebase**: Auth/Firestore 연동 유지 (소셜 기능)
- **기존 데이터 호환**: FocusSession, BucketSkin enum 등 기존 모델과의 마이그레이션 고려
- **다크/라이트 모드**: AppColors 토큰 시스템 유지 및 확장

## 5. 기존 유지 사항

- MVVM + DI Container 아키텍처
- 무한 모드 (Infinity Mode)
- 집중 확인 알림 + 10초 타임아웃 → 자동 일시정지
- 메뉴바 타이머 상태/제어
- 소셜 기능 (Feature Flag 토글)
- deploy.sh 빌드/배포 파이프라인
- Homebrew Tap 배포

## 6. 요청 사항

위 내용을 바탕으로 RainDrop 2.0의 UI/UX를 설계해 주세요.

### 산출물
1. **화면별 상세 와이어프레임 설명** — 각 화면의 레이아웃, 컴포넌트, 인터랙션
2. **디자인 토큰 제안** — 색상, 타이포그래피, 간격, 애니메이션 스펙
3. **인터랙션 플로우** — 사용자 시나리오별 흐름 (첫 사용, 일상 세션, 회고, 보상 확인)
4. **애니메이션 스펙** — 물, 비, 구름, 전환 등 주요 모션의 타이밍/이징/트리거
5. **마이그레이션 고려사항** — 기존 v1.x 사용자의 데이터·경험 전환 방안

### 중요
- "채움으로 삶의 밀도를 기록한다"가 모든 설계 결정의 기준
- 과하지 않게. 고요한 아름다움. 계속 켜두고 싶은 화면
- macOS 앱답게. 네이티브 느낌을 유지하되, 감성적 깊이 추가
- 실제 SwiftUI로 구현 가능한 수준의 설계
